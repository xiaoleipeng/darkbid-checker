# Word暗标编制要求检查工具 - 设计文档

## 1. 项目背景

投标文件技术标（暗标）部分需按照严格的格式要求编制，不符合要求将被否决投标。手动逐项检查耗时且易遗漏，需要一个自动化工具来检测不符合项并辅助修正。

## 2. 需求概述

给定一个 `.docx` 格式的Word文档，工具需要：
1. 自动检测所有不符合暗标编制要求的内容
2. 以列表形式标注每个不符合项（类别、描述、位置）
3. 提供交互界面，用户可逐项确认或取消修改
4. 将确认的修改应用到文档并另存为新文件

## 3. 检查规则

### 3.1 版面要求
| 规则 | 要求值 | 检查方式 |
|------|--------|----------|
| 纸张大小 | A4 (21.0cm × 29.7cm) | 读取 section.page_width/page_height |

### 3.2 颜色要求
| 规则 | 要求值 | 检查方式 |
|------|--------|----------|
| 文字颜色 | 黑色 (#000000) | 遍历所有 run，检查 font.color.rgb |

### 3.3 字体要求
| 规则 | 要求值 | 检查方式 | 备注 |
|------|--------|----------|------|
| 字体名称 | 宋体 | font.name + eastAsia 属性 | 表格内不检查 |
| 字号 | 四号 (14pt) | font.size | 表格内不检查 |
| 字形 | 常规 | font.bold/italic/underline 均为 False | — |
| 标点 | 中文标点 | 扫描正文中的英文标点字符，排除数字中的小数点/千分位 | 可自动替换 |

### 3.4 排版要求
| 规则 | 要求值 | 检查方式 |
|------|--------|----------|
| 上边距 | 2.5cm | section.top_margin |
| 下/左/右边距 | 2.0cm | section.bottom/left/right_margin |
| 目录 | 不得设置 | 检测文档中的 TOC 域代码 |
| 行间距 | 固定值30磅 | paragraph_format.line_spacing_rule + line_spacing |
| 对齐方式 | 左对齐 | paragraph_format.alignment |
| 首行缩进 | 2字符 (≈28pt) | paragraph_format.first_line_indent |
| 段前段后 | 0 | space_before / space_after |
| 页眉 | 无 | section.header 内容为空 |
| 页脚 | 无 | section.footer 内容为空 |
| 页码 | 无 | footer 中无 PAGE 字段 |
| 开头空格 | 不允许 | 段落文本不以空格开头 |

### 3.5 封面目录要求
| 规则 | 要求值 | 检查方式 |
|------|--------|----------|
| 目录 | 不得存在 | 扫描 TOC 域代码 (w:instrText 含 "TOC") |
| 封面 | 不得存在 | 暂由用户自行确认 |

### 3.6 身份信息要求
| 规则 | 要求值 | 检查方式 |
|------|--------|----------|
| 投标人名称 | 不得出现 | 用户配置关键词，全文扫描匹配 |
| 人员名称 | 不得出现 | 同上 |
| 微标/Logo | 不得出现 | 暂无法自动检测图片内容 |

### 3.7 其他要求
| 规则 | 要求值 | 检查方式 |
|------|--------|----------|
| 页数 | ≤300页 | 按段落数估算（python-docx 无法精确获取页数） |

### 3.8 检查排除范围
- 表格内的文字：字体和字号不做检查（规则原文："图、表内的字体及字号不作要求"）
- 空白段落：跳过

## 5. 核心数据结构

```python
class Issue:
    description: str    # 问题描述
    location: str       # 位置信息（段落/节）
    category: str       # 分类（版面/字体/排版/颜色/其他）
    fix_func: Callable  # 修复函数（None 表示无法自动修复）
    selected: bool      # 用户是否选中修复
```

## 6. 交互流程

```
用户选择文件 → 执行全部检查 → 展示问题列表
                                    ↓
                    用户逐项 确认/取消 修改
                                    ↓
                    点击"应用修改" → 执行选中的修复
                                    ↓
                    选择保存路径 → 另存为新文件
                                    ↓
                    重新检查修改后文件 → 展示剩余问题
```

## 7. 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| 文档解析 | python-docx | Python生态最成熟的docx库 |
| GUI框架 | tkinter | Python内置，无需额外安装 |
| Python版本 | 3.8+ | 兼容性好 |

## 8. 使用方式

```bash
# GUI模式（交互式，含关键词输入框）
python3 check_word.py

# CLI模式（仅报告）
python3 check_word.py <文件路径>

# CLI模式 + 身份信息关键词检测
python3 check_word.py <文件路径> --keywords "公司名称,张三,李四"
```

## 9. 已知限制

1. **页数统计**：python-docx 无法获取精确页数，仅通过段落数估算
2. **封面检测**：无法程序化判断是否存在封面页（无明确的结构标记），需用户确认
3. **图片/Logo检测**：无法自动识别图片中是否含有投标人标识
4. **中文标点**：数字中的英文小数点和千分位逗号已排除，但特殊场景（如英文缩写）可能误报
5. **样式继承**：当 run 未显式设置字体/字号时（继承自样式），python-docx 返回 None，不会报错
